# VLog 博客系统

## 项目介绍

VLog 是一个轻量级博客社区示例项目，后端使用 Node.js + Express 提供接口，前端使用单个 HTML 文件实现页面渲染与交互，整体风格参考了经典论坛页面。项目支持文章列表、搜索、分页、详情查看、发布、编辑和删除等基本功能，适合作为入门级全栈练习或本地演示项目。

这个项目的特点是结构非常直接：

- 服务端负责静态资源托管和文章接口
- 前端负责页面切换、表单提交和列表渲染
- 数据直接保存在本地 JSON 文件中，不依赖数据库

## 技术栈

- Node.js
- Express 4
- 原生 HTML、CSS、JavaScript
- JSON 文件存储

## 后端代码解析

### 服务初始化

服务入口在 [server.js](server.js)。启动时会做三件事：

1. 创建 Express 应用并监听 3000 端口
2. 检查 data 目录和 posts.json 是否存在，不存在则自动初始化
3. 挂载 JSON 解析中间件、表单解析中间件和静态文件目录

这部分设计说明项目具备“开箱即用”的特点，首次运行不需要手动准备数据文件。

### 数据读写

项目没有接入数据库，而是通过两个函数完成数据持久化：

- `readPosts()`：同步读取 [data/posts.json](data/posts.json) 并解析为数组
- `writePosts(posts)`：将文章数组格式化写回 JSON 文件

这种方式实现简单，适合教学和原型验证，但不适合高并发场景，因为它依赖同步文件 IO。

### API 设计

#### 1. 获取文章列表

`GET /api/posts`

支持的查询参数：

- `page`：页码，默认 1
- `pageSize`：每页数量，默认 15，最大 50
- `keyword`：关键字搜索，匹配标题和内容

处理逻辑：

- 按 `updatedAt` 倒序排列
- 根据关键字过滤文章
- 返回分页后的简化数据
- 自动生成摘要 `summary`

#### 2. 获取文章详情

`GET /api/posts/:id`

处理逻辑：

- 按 ID 查找文章
- 若不存在则返回 404
- 若存在则浏览量 `views + 1`
- 将更新后的结果重新写入 JSON 文件

#### 3. 发布文章

`POST /api/posts`

请求体字段：

- `title`
- `content`
- `author`

处理逻辑：

- 校验标题和内容不能为空
- 使用 `crypto.randomUUID()` 生成唯一 ID
- 自动补充创建时间、更新时间和初始浏览量
- 作者为空时默认写为“匿名”

#### 4. 修改文章

`PUT /api/posts/:id`

处理逻辑：

- 校验标题和内容
- 查找目标文章
- 更新标题、作者、内容和 `updatedAt`

#### 5. 删除文章

`DELETE /api/posts/:id`

处理逻辑：

- 根据 ID 定位文章
- 删除对应记录
- 重新写回数据文件

### SPA 回退路由

最后一个 `app.get('*')` 会将所有非 API 请求都返回 [public/index.html](public/index.html)。

这意味着项目采用的是一个简单的单页应用思路：

- API 请求由后端处理
- 页面路由和视图切换由前端脚本控制

## 前端代码解析

前端代码全部写在 [public/index.html](public/index.html) 中，包含样式、页面结构和脚本逻辑。

### 页面风格

CSS 使用了经典论坛风格的视觉设计，包括：

- 顶部导航栏
- 蓝色论坛式头部
- 表格式帖子列表
- 面板式内容区
- 自定义消息弹窗

这种实现方式没有引入任何前端框架，所有交互都通过原生 JavaScript 完成。

### 前端状态管理

脚本维护了三个全局状态变量：

- `currentPage`：当前页码
- `currentKeyword`：当前搜索关键字
- `currentView`：当前视图类型，取值为 `list`、`detail`、`editor`

这套状态足以支撑这个单页应用的页面切换。

### 核心前端函数

#### `loadList(page, keyword)`

负责加载文章列表页面：

- 调用 `/api/posts`
- 渲染列表表格
- 生成分页按钮
- 根据关键字显示搜索结果标题

#### `viewPost(id)`

负责加载文章详情：

- 调用 `/api/posts/:id`
- 展示标题、作者、时间、浏览量和正文
- 提供编辑和删除入口

#### `showEditor(editId)`

负责展示发帖或编辑页面：

- 无 `editId` 时为新建文章
- 有 `editId` 时先加载原文章内容，再填充表单

#### `submitPost(editId)`

负责提交表单：

- 新建时调用 `POST /api/posts`
- 编辑时调用 `PUT /api/posts/:id`
- 成功后跳转到详情页或列表页

#### `deletePost(id)`

负责删除文章：

- 先弹出确认框
- 调用 `DELETE /api/posts/:id`
- 成功后返回列表页

### 安全处理

前端对标题、作者、提示框内容使用了 `escapeHtml()` 进行转义，用来减少直接插入 HTML 带来的风险。正文在详情页里也进行了字符转义，并将换行转换为 `<br>`，因此默认不会直接执行用户输入的 HTML。

## 数据流说明

项目的数据流很清晰：

1. 用户在浏览器中进行搜索、发帖、编辑或删除操作
2. 前端通过 `fetch` 调用后端接口
3. Express 路由读取或改写 [data/posts.json](data/posts.json)
4. 服务端返回 JSON 响应
5. 前端根据响应结果重新渲染页面

这是一种典型的小型 CRUD 应用结构。

## 返回数据格式

接口统一采用类似结构：

json
{
  "code": 0,
  "data": {},
  "msg": "操作成功"
}

其中：

- `code = 0` 表示成功
- `code = 1` 表示业务失败
- `msg` 表示提示信息

## 适用场景

这个项目适合以下用途：

- Node.js + Express 入门练习
- 原生 JavaScript 前后端交互示例
- 轻量博客或论坛原型演示
- 文件型存储 CRUD 项目教学

## 当前实现的优点

- 项目结构简单，容易理解
- 启动成本低，不依赖数据库
- 前后端职责划分清晰
- API 设计完整，覆盖常见博客操作
- 页面交互完整，具备实际可用性

## 当前实现的局限

- 数据存储依赖本地 JSON 文件，不适合多人并发写入
- 文件读写使用同步 IO，在数据量大时性能会受影响
- 没有登录、权限控制和身份校验
- 没有富文本编辑器、图片上传和评论系统
- 没有前端构建流程和模块化拆分

## 可继续优化的方向

1. 将 JSON 存储替换为 SQLite、MySQL 或 MongoDB
2. 为文章增加分类、标签、评论和点赞功能
3. 引入用户登录和后台管理
4. 将前端拆分为独立 JS 模块或迁移到 Vue / React
5. 增加输入校验、错误处理和日志系统
6. 补充测试和部署说明

## 总结

VLog 是一个典型的小型全栈 CRUD 项目。它通过极少的文件完成了一个博客系统的核心功能，代码路径短、逻辑直接，很适合用来学习以下内容：

- Express 路由设计
- 原生前端与后端接口联动
- JSON 文件持久化
- 单页应用式页面切换
